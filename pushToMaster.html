The first thing you should know!

You should not push to master

For these reasons:

  1. Master branch is probably continuously deployed, if we push straight to master we are prone to get bugs in our final version.
  2. Your code will not be reviewed
  3. You can not easily undo the changes
  4. Nobody will know exactly what you changed
  5. Your colleagues will just have to pray you didn't break anything.
  6. If your code created bugs, some users will have a bad experience.
